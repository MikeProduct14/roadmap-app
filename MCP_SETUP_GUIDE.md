## 📋 Настроенные MCP серверы

### 1. context7
- **Команда:** `uvx context7`
- **Назначение:** Работа с контекстом и документацией
- **Статус:** Готов к использованию

### 2. supabase
- **Команда:** `uvx mcp-server-supabase`
- **Назначение:** Работа с Supabase базой данных
- **Статус:** Готов к использованию

### 3. playwright
- **Команда:** `uvx mcp-playwright`
- **Назначение:** Автоматизация браузерного тестирования
- **Статус:** Готов к использованию

## 🔧 Как использовать

### Активация серверов в Kiro

1. **Перезапусти Kiro** чтобы серверы подключились автоматически

2. **Или используй команду:**
   - Открой палитру команд (Ctrl+Shift+P)
   - Найди "MCP: Reconnect Servers"
   - Выполни команду

3. **Или через панель:**
   - Открой панель "MCP Server" в Kiro
   - Нажми кнопку переподключения

### Проверка работы

После перезапуска Kiro или переподключения серверов:
- Серверы автоматически загрузятся через `uvx`
- При первом запуске `uvx` скачает необходимые пакеты
- Это может занять несколько секунд

## ⚠️ Важно: Настройка PATH

Чтобы `uv` и `uvx` работали постоянно, добавь путь в системную переменную PATH:

**Через PowerShell (рекомендуется):**
```powershell
[Environment]::SetEnvironmentVariable("Path", "$env:Path;C:\Users\Mike\.local\bin", "User")
```

После этого перезапусти Kiro.

## 📝 Конфигурация

Файл: `C:\Users\Mike\.kiro\settings\mcp.json`

```json
{
  "mcpServers": {
    "context7": {
      "command": "uvx",

      "disabled": false

    "supabase": {
      "command": "uvx",

      "disabled": false

    "playwright": {

      "disabled": false
    }
  }

```
